# Security Policy

## Reporting Security Issues

**DO NOT** open a public GitHub issue for security vulnerabilities.

Instead, please email: security@prism.dev

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will acknowledge receipt within 24 hours and provide updates on progress.

## Scope

Security policy applies to:
- PRISM methodology and documentation
- ETPD skill definition
- Code examples and templates
- Tools and scripts

## Disclosure Timeline

- 0 days: You report the issue
- 24 hours: We acknowledge receipt
- 7 days: We provide initial assessment
- 30 days: We release a fix
- 30+ days: Public disclosure with your approval

Thank you for helping keep PRISM secure!
